# PPTX 课件生成子技能

## 两种触发场景

**场景 A — 直接生成 PPTX**：教师明确要求 PPTX 格式。
**场景 B — 从 HTML 转换**：HTML 课件生成后，教师确认需要 PPTX 版本。

---

## 场景 A：直接生成 PPTX

### Step 1 — 确认参数

同 ppt-html 子技能，收集：学科、年级、课题、教材版本、主题色。

### Step 2 — 生成 PPTX

使用 `pptx` skill 的能力，按以下规格生成：

**视觉规格（专为支教适配）：**
- 幻灯片大小：16:9（33.87cm × 19.05cm）
- 标题字号：≥ 36pt（相当于屏幕上 ~51px）
- 正文字号：≥ 24pt
- 重点关键词：红色底色白色字，粗体
- 配色：简洁高对比，避免渐变阴影（兼容老版 PowerPoint）
- 每页内容原则：同 HTML 版，聚焦一个核心知识点，可根据内容自适应页数

**PPTX 特殊处理：**
- 动效使用 PowerPoint 原生动画（进入效果：飞入、弹跳、缩放）
- 不使用 WordArt 或过于复杂的图形，确保在低版本 Office 下可用
- 图片使用 PNG/JPEG 内嵌（不使用 OLE 对象）
- 字体选择系统通用字体：微软雅黑/宋体（中文），Calibri（英文）

### Step 3 — 输出

输出路径：`[课题名称]-课件.pptx`

---

## 场景 B：HTML 转 PPTX

### 说明损失

在开始转换前，告知教师：

> "PPTX 转换会尽量保留原课件的视觉效果，但以下内容无法完整保留：
> - WebGL 粒子背景 → 转为静态渐变背景
> - Canvas confetti 特效 → 转为 PowerPoint 原生'彩纸'动画
> - 3D 卡片翻转 → 转为'飞入'进入动画
> - 其他 CSS 动效 → 转为最接近的 PPT 动画
> 文字内容、布局结构和配色方案将完整保留。"

### Step 1 — 解析 HTML 内容

从传入的 HTML 课件中提取每页内容：
- `<h1>` / `<h2>` → 幻灯片标题
- `<p>` / `<ul>` / `<ol>` → 正文内容
- inline `<svg>` → 导出为 PNG（如不能导出则用最接近的 Unicode emoji 替代）
- `<div class="bilingual-bar">` → 保留为 PPTX 文本框，置于页面底部
- `data-anim` 属性 → 映射到 PPT 动画效果

### Step 2 — 生成 PPTX

按解析内容逐页生成，视觉规格同场景 A。

### Step 3 — 输出

输出路径：`[课题名称]-课件-pptx版.pptx`，同时告知教师两个文件的区别。
